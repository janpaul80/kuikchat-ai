## Non-Negotiable Rules

### Code Quality
- **No fake data** — placeholder rows must be labeled honestly as "coming soon" or "no data yet"
- **No fake device frames** — show real screenshots only
- **No fake counts** — no inflated numbers, no fake "connected" states
- **Real RLS** — every business feature uses Row Level Security with owner-only writes
- **Real proofs required** — every feature needs a screenshot of the actual screen (not a mockup)

### Deployment (Mandatory)
- **Every change must deploy** — no feature is "done" until it runs on production
- **Push to GitHub first** — branch must be pushed before deployment
- **Zero-downtime deployment** — use the shadow build pattern (`/opt/kuikchat_build` → swap)
- **Systemd service** — restart `kuikchat.service`, not manual PM2

### Git Workflow
- **Feature branch per slice** — naming: `slice/2c-business-tools-hub`
- **PR workflow** — create PR, verify, then merge
- **Never push directly to main** — except hotfixes

---

## Slice Workflow

Each slice follows this exact sequence:

1. Check approved spec in `docs/`
2. Create feature branch: `slice/{id}-{name}`
3. Build the feature
4. Add database migrations if needed
5. Push branch to GitHub
6. Deploy to production server
7. Take real screenshot
8. Mark as "ready for review"
